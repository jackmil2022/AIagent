# 🎯 Prompt工程：基础技巧

> **学会和AI说话的艺术，让大模型为你所用**

---

## 📝 前言：什么是Prompt？

想象一下，你有一个超级聪明但完全不了解你需求的助手。你说"帮我写个东西"，它完全不知道你想写什么。但如果你说"帮我写一封礼貌的商务邮件，主题是拒绝合作邀请"，它就能精准地完成任务。

**Prompt（提示词）就是你给AI的指令。**

Prompt工程，就是学习如何写出让AI准确理解你意图的指令。

---

## 🔰 1. 零样本学习 (Zero-shot)

### 1.1 什么是零样本？

零样本就是**不给任何示例**，直接让AI完成任务。

### 1.2 生活例子

想象你问朋友："今天天气怎么样？" 你没有给任何例子，直接问。这就是零样本。

### 1.3 代码示例

```python
# 零样本分类
prompt = """
请判断以下评论的情感是正面还是负面：

"这家餐厅的菜太好吃了，服务也很棒！"
"""

# AI会直接回答：正面
```

### 1.4 适用场景
- 简单的分类任务
- 常见的问答
- 简单的翻译

---

## 🔰 2. 少样本学习 (Few-shot)

### 2.1 什么是少样本？

少样本就是**给几个示例**，让AI学习模式后完成任务。

### 2.2 生活例子

想象你教小朋友认水果：
- "这是苹果，红色的，圆圆的"
- "这是香蕉，黄色的，弯弯的"
- 然后你拿一个橘子问："这是什么？"

小朋友通过前面的例子学会了如何识别水果。

### 2.3 代码示例

```python
# 少样本分类
prompt = """
请判断评论的情感：

评论：服务态度很差
情感：负面

评论：菜品味道不错
情感：正面

评论：等位时间太长了
情感：负面

评论：环境很舒适，价格也实惠
情感："""

# AI会回答：正面
```

### 2.4 最佳实践

| 技巧 | 说明 |
|------|------|
| 示例数量 | 3-5个最佳 |
| 示例多样性 | 覆盖不同类型 |
| 示例顺序 | 最相关的放最后 |
| 格式一致 | 保持输入输出格式统一 |

## 🔰 3. 思维链 (Chain-of-Thought, CoT)

### 3.1 什么是思维链？

思维链就是**让AI展示推理过程**，而不是直接给答案。

### 3.2 生活例子

想象你做数学题：
- 普通回答："答案是42"
- 思维链："首先计算5×6=30，然后30+12=42，所以答案是42"

思维链让你能看到AI的思考过程，更容易发现错误。

### 3.3 代码示例

```python
# 普通回答
prompt1 = """
小明有5个苹果，小红有3个苹果，他们一共有多少个苹果？
"""

# 思维链回答
prompt2 = """
小明有5个苹果，小红有3个苹果，他们一共有多少个苹果？

请一步步思考：
"""

# AI会回答：
# 1. 小明有5个苹果
# 2. 小红有3个苹果
# 3. 5 + 3 = 8
# 4. 所以他们一共有8个苹果
```

### 3.4 什么时候用CoT？

| 场景 | 是否用CoT |
|------|-----------|
| 简单问答 | ❌ 不需要 |
| 数学计算 | ✅ 强烈推荐 |
| 逻辑推理 | ✅ 强烈推荐 |
| 复杂分析 | ✅ 推荐 |

## 🔰 4. 零样本思维链 (Zero-shot CoT)

### 4.1 什么是零样本思维链？

在零样本的基础上，加上"让我们一步步思考"这样的提示。

### 4.2 代码示例

```python
# 零样本思维链
prompt = """
一个商店打8折促销，原价100元的商品现在多少钱？

让我们一步步思考：
"""

# AI会自动展示推理过程
```

### 4.3 万能咒语

```
让我们一步步思考。
请展示你的推理过程。
首先...然后...最后...
```

## 🔰 5. 角色扮演 (Role Playing)

### 5.1 什么是角色扮演？

给AI设定一个**特定角色**，让它以该角色的身份回答问题。

### 5.2 生活例子

想象你在玩角色扮演游戏：
- 你扮演医生，我扮演病人
- 你扮演老师，我扮演学生

角色扮演让AI更专业、更一致。

### 5.3 代码示例

```python
# 角色扮演
prompt = """
你是一位经验丰富的Python编程老师，擅长用简单易懂的方式解释复杂概念。

请解释什么是面向对象编程？
"""

# AI会用老师的口吻，更耐心地解释
```

### 5.4 常用角色

| 角色 | 适用场景 |
|------|----------|
| 编程老师 | 解释技术概念 |
| 产品经理 | 分析需求 |
| 代码审查者 | 找bug |
| 技术面试官 | 准备面试 |
| 翻译专家 | 高质量翻译 |

## 🔰 6. 输出格式控制

### 6.1 为什么控制输出格式？

- 便于解析
- 结构化数据
- 后续处理

### 6.2 常用格式

#### JSON格式
```python
prompt = """
请以JSON格式返回以下信息：

姓名：张三
年龄：25岁
职业：程序员

输出格式：
{
  "name": "姓名",
  "age": 年龄,
  "occupation": "职业"
}
"""
```

#### 表格格式
```python
prompt = """
请用表格对比Python和JavaScript：

| 特性 | Python | JavaScript |
|------|--------|------------|
"""
```

#### Markdown格式
```python
prompt = """
请用Markdown格式总结以下内容：
...
"""
```

## 🔰 7. 分隔符与结构化

### 7.1 使用分隔符

分隔符帮助AI区分指令和内容：

```python
prompt = """
请总结以下文章：

---
[文章内容]
---

总结：
"""
```

### 7.2 常用分隔符

| 分隔符 | 使用场景 |
|--------|----------|
| `---` | 分隔不同部分 |
| `"""` | 包裹长文本 |
| `<tag>` | XML风格 |
| `###` | Markdown标题 |

---

## 🔰 8. 指令明确化

### 8.1 避免模糊指令

```python
# ❌ 模糊的指令
"写点东西"

# ✅ 明确的指令
"写一篇300字的关于人工智能发展趋势的文章，面向非技术读者"
```

### 8.2 指令清单

| 要素 | 说明 | 示例 |
|------|------|------|
| 任务 | 做什么 | 总结、分类、翻译 |
| 输入 | 处理什么 | 文章、评论、代码 |
| 输出 | 得到什么 | 列表、表格、JSON |
| 约束 | 限制条件 | 字数、风格、格式 |
