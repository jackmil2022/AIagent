# 束搜索 (Beam Search)

> 来源：李沐《动手学深度学习》

---

## 1. 什么是束搜索？

### 1.1 贪心搜索的问题

贪心搜索每一步只选概率最高的词，但可能错过全局最优解：

```
句子1: P("我" → "喜欢" → "你") = 0.4
句子2: P("我" → "爱" → "你") = 0.5  (局部最高)

贪心选择句子2，但句子1的联合概率可能更高
```

### 1.2 束搜索的思想

**保留 k 个最优候选（束宽 beam width）**

```
束宽 k=2 时：
第1步: 保留概率最高的2个词
第2步: 从这2个词扩展，保留最高的2个序列
...继续直到结束
```

---

## 2. 算法步骤

### 2.1 基本流程

```python
def beam_search(model, input_seq, beam_width=3, max_len=50):
    """
    beam_width: 束宽，保留的候选数量
    """
    # 初始状态
    beams = [(input_seq, 0.0)]  # (序列, 对数概率)
    
    for step in range(max_len):
        all_candidates = []
        
        for seq, score in beams:
            # 如果序列已结束，直接保留
            if seq[-1] == EOS_TOKEN:
                all_candidates.append((seq, score))
                continue
            
            # 获取下一个词的概率分布
            logits = model(seq)
            probs = softmax(logits)
            
            # 保留 top-k 个候选
            top_k = torch.topk(probs, beam_width)
            for prob, idx in zip(top_k.values, top_k.indices):
                new_seq = seq + [idx]
                new_score = score + log(prob)
                all_candidates.append((new_seq, new_score))
        
        # 选择全局 top-k
        beams = sorted(all_candidates, key=lambda x: x[1], reverse=True)[:beam_width]
    
    return beams[0]  # 返回得分最高的序列
```

## 3. 优缺点

| 优点 | 缺点 |
|------|------|
| 比贪心搜索质量高 | 计算量比贪心大 |
| 找到近似最优解 | 束宽大时内存消耗高 |
| 广泛用于翻译、摘要 | 可能漏掉全局最优 |

---

## 4. 实际应用

| 应用 | 典型束宽 |
|------|----------|
| 机器翻译 | 4-5 |
| 文本生成 | 3-5 |
| 语音识别 | 10-20 |
