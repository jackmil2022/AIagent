# 单机多卡并行

> 来源：李沐《动手学深度学习》

---

## 1. 为什么需要多卡训练？

### 1.1 单卡的限制

- **显存不足**：大模型放不进单张 GPU
- **训练太慢**：数据量太大，单卡训练需要数周

### 1.2 多卡的优势

```
4张 GPU 并行：
- 显存扩大 4 倍
- 训练速度提升 3-4 倍
```

---

## 2. 数据并行 (Data Parallelism)

### 2.1 核心思想

**每张卡都有完整模型副本，但处理不同数据**

```
数据集 → 分成 4 份
    ↓
GPU 1: 模型 + 数据1 → 梯度1
GPU 2: 模型 + 数据2 → 梯度2
GPU 3: 模型 + 数据3 → 梯度3
GPU 4: 模型 + 数据4 → 梯度4
    ↓
合并梯度 → 更新所有模型
```

### 2.2 PyTorch 实现

```python
import torch
import torch.nn as nn
from torch.nn.parallel import DataParallel, DistributedDataParallel

# 基础数据并行
model = nn.Linear(10, 2)
model = DataParallel(model)  # 自动使用所有 GPU

# 输入数据
x = torch.randn(32, 10)  # batch_size=32
output = model(x)  # 自动分配到多卡
```

### 2.3 分布式数据并行 (DDP)

```python
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

# 初始化进程组
dist.init_process_group(backend='nccl')

# 包装模型
model = DDP(model.cuda(), device_ids=[local_rank])

# 使用 DistributedSampler
from torch.utils.data import DataLoader, DistributedSampler
```

## 3. 模型并行 (Model Parallelism)

### 3.1 核心思想

**将模型切分到多张 GPU**

```
GPU 1: 模型前半部分
GPU 2: 模型后半部分

输入 → GPU1处理 → 结果传给GPU2 → 最终输出
```

### 3.2 适用场景

- 模型太大，单卡放不下
- 需要切分层数

## 4. 实战技巧

### 4.1 混合精度训练

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()
    
    with autocast():  # 自动使用 FP16
        output = model(data)
        loss = criterion(output, target)
    
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

### 4.2 梯度累积

```python
accumulation_steps = 4

for i, (data, target) in enumerate(dataloader):
    output = model(data)
    loss = criterion(output, target) / accumulation_steps
    loss.backward()
    
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

## 5. 性能对比

| 方法 | 显存 | 速度 | 适用场景 |
|------|------|------|----------|
| 单卡 | 1x | 1x | 小模型 |
| DataParallel | 1x | 2-3x | 模型能放下 |
| DDP | 1x | 3-4x | 多卡训练 |
| 模型并行 | 1x/卡 | 1-2x | 超大模型 |
