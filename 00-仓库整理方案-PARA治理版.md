# 仓库整理方案

这个仓库已经有主干目录，但现在的核心问题不是“没有分类”，而是“分类口径混用”：

1. `01-基础准备`、`02-机器学习` 同时混放主线教材、中文改写稿、原文镜像。
2. `03-深度学习` 到 `06-行业应用` 更像课程主线，而 `90-资料库` 又没有承接大部分原文资料。
3. 顶层索引已经存在，但没有把“主学习路径”和“参考资料仓”彻底分开。

## 当前判断

- 这是一个学习资料仓，不是代码仓。
- 现有编号主线是对的，不该推倒重来。
- 最大噪音来自大批 `源文件/GitHub原文/`、课程镜像、重复导出内容。
- 先立规则，再搬文件；否则会把现有 Obsidian 链接打断。

## 现状摘要

- 文件总量约 `37000`
- 顶层主分区：
  - `01-基础准备`
  - `02-机器学习`
  - `03-深度学习`
  - `04-大模型与NLP`
  - `05-Agent开发`
  - `06-行业应用`
  - `90-资料库`
  - `91-项目实战`
  - `99-归档`
- 体量最大的目录：
  - `02-机器学习` 约 `19742` 文件
  - `01-基础准备` 约 `16638` 文件
  - `03-深度学习` 约 `463` 文件

结论很直接：仓库的“重”主要不在主线课程，而在镜像资料。

## 建议的治理规则

按 PARA 的思路，这个仓库用最小改造即可：

### 1. 主线区

保留现在的编号结构，作为真正要学、要看的内容：

- `00-*`：总入口、总路线、导航
- `01-基础准备`
- `02-机器学习`
- `03-深度学习`
- `04-大模型与NLP`
- `05-Agent开发`
- `06-行业应用`
- `91-项目实战`

这些目录里只放三类内容：

1. 课程索引
2. 中文整理稿
3. 必要的小型示例代码或数据

### 2. 资料区

`90-资料库` 统一承接“参考但不属于主线”的内容：

- GitHub 原文镜像
- 外部课程原始资料
- 历史报告
- 大型素材
- 第三方 notebook 原件

建议在 `90-资料库` 下固定四层：

```text
90-资料库/
├── 01-GitHub原文
├── 02-课程原始资料
├── 03-研究报告
└── 04-素材资源
```

### 3. 归档区

`99-归档` 只放：

- 废弃目录
- 被新版本替代的旧索引
- 已迁移但不想立刻删除的内容

不要把还在使用的东西扔进归档。

## 目录判定标准

后面整理时，每个文件只问一个问题：

“这是主线学习内容，还是参考资料原件？”

- 如果是你会直接阅读、跟学、复习的内容，留在 `01-06` / `91`
- 如果只是来源、备份、镜像、附件，进 `90-资料库`

这个标准比按文件类型分更稳。`.ipynb` 不一定是主线，`.md` 也不一定是主线。

## 最小落地方案

不建议一次性大搬仓库。按下面顺序做。

### Phase 1：只整理索引，不搬文件

目标：

- 明确每个顶层目录的职责
- 在总索引里补一条规则：`90-资料库` 是参考区，不是起点
- 给 `01`、`02`、`03` 分别补一个“本目录包含什么 / 不包含什么”

这是风险最低的一步。

### Phase 2：先搬最重的镜像资料

优先处理：

1. `90-资料库/01-GitHub原文/基础准备`
2. `90-资料库/01-GitHub原文/机器学习`

目标位置：

```text
90-资料库/01-GitHub原文/基础准备/
90-资料库/01-GitHub原文/机器学习/
```

同时在原位置留一个简短索引说明，不留大批原文件。

### Phase 3：清理重复课程副本

目前 `03-深度学习` 下存在“课程整理版”和“源文件课程版”并存的情况。这里不该全删，应该二选一：

- 主学习版保留在 `03-深度学习`
- 原始课程副本迁到 `90-资料库/02-课程原始资料`

### Phase 4：归档失效入口

搬完以后，把旧入口说明、旧目录副本、废弃索引放到 `99-归档`。

## 这次整理建议不要做的事

- 不要全仓重命名
- 不要一次性移动上万文件
- 不要先改 Obsidian 全部双链
- 不要按扩展名机械分类

这几件事工作量大，收益低，还容易把现有引用打碎。

## 我给你的建议结论

这个仓库最合适的整理方向不是“重做结构”，而是：

1. 保留现有编号主线
2. 把主线内容和原始资料彻底分层
3. 先整理索引，再分批搬最重目录

## 推荐的下一步

下一步只做一件事就够：

`把 01/02 里的 GitHub 原文镜像迁到 90-资料库，并回填索引页。`

这是最小但最有效的一刀，能立刻把仓库噪音降下来。
