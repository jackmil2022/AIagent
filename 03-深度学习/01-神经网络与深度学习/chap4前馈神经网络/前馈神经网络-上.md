# 前馈神经网络（上）激活函数 + 两层 MLP 二分类

本节做三件事：

1. 看一下常用**激活函数**的形状（sigmoid / tanh / ReLU 族）。
2. 用 PyTorch tensor 手写一个两层 MLP 的**前向 + 反向**，再用 `autograd` 跑一遍同一个网络，验证梯度逐元素一致——感受 autograd 在做什么。
3. 用 `nn.Module` 在 moons 数据集上完成二分类，画决策边界。

## 1. 激活函数

PyTorch 把激活函数放在 `torch.nn.functional` 里（也有 `nn.ReLU` 等 module 形式）。

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
import matplotlib.pyplot as plt

torch.manual_seed(0)

x = torch.linspace(-6, 6, 400)
acts = {
    'sigmoid':     torch.sigmoid(x),
    'tanh':        torch.tanh(x),
    'relu':        F.relu(x),
    'leaky_relu':  F.leaky_relu(x, negative_slope=0.1),
    'elu':         F.elu(x),
    'softplus':    F.softplus(x),
}

plt.figure(figsize=(9, 5))
for name, y in acts.items():
    plt.plot(x.numpy(), y.numpy(), label=name)
plt.axhline(0, c='gray', lw=0.5); plt.axvline(0, c='gray', lw=0.5)
plt.legend(); plt.title('activation functions'); plt.grid(alpha=0.3); plt.show()
```

### 激活函数的导数

训练时反向传播需要激活函数的导数。`autograd` 帮我们算，但提前对照一下能加深理解。

```python
x = torch.linspace(-6, 6, 400, requires_grad=True)
fig, axes = plt.subplots(2, 3, figsize=(12, 6))
for ax, (name, fn) in zip(axes.flat, [
    ('sigmoid', torch.sigmoid), ('tanh', torch.tanh),
    ('relu', F.relu), ('leaky_relu', lambda t: F.leaky_relu(t, 0.1)),
    ('elu', F.elu), ('softplus', F.softplus),
]):
    y = fn(x)
    grad = torch.autograd.grad(y.sum(), x, create_graph=False)[0]
    ax.plot(x.detach().numpy(), y.detach().numpy(), label=name)
    ax.plot(x.detach().numpy(), grad.detach().numpy(), '--', label=f"d{name}/dx")
    ax.axhline(0, c='gray', lw=0.5); ax.legend(); ax.grid(alpha=0.3)
plt.tight_layout(); plt.show()
```

## 2. 两层 MLP：手算反向 vs autograd

结构：$\boldsymbol{x} \xrightarrow{W_1, b_1} \boldsymbol{z}_1 \xrightarrow{\text{Logistic}} \boldsymbol{a}_1 \xrightarrow{W_2, b_2} z_2 \xrightarrow{\text{Logistic}} \hat y$。

二分类交叉熵损失：$\mathcal{L} = -[y \log \hat y + (1-y) \log (1-\hat y)]$。

对单个样本逐步手算 $\partial \mathcal{L} / \partial W_1, b_1, W_2, b_2$，然后让 autograd 算同一组梯度，验证逐元素一致。

```python
torch.manual_seed(0)

# 一个样本
x = torch.randn(3)
y = torch.tensor(1.0)

# 参数（先 detach，因为我们要手算，autograd 部分再单独跑）
W1 = torch.randn(4, 3); b1 = torch.zeros(4)
W2 = torch.randn(4);    b2 = torch.zeros(())

# ---- 前向 ----
z1 = W1 @ x + b1
a1 = torch.sigmoid(z1)
z2 = W2 @ a1 + b2
a2 = torch.sigmoid(z2)
loss = -(y * torch.log(a2) + (1 - y) * torch.log(1 - a2))
print(f'forward:  z1.shape={list(z1.shape)}  a2={a2.item():.4f}  loss={loss.item():.4f}')

# ---- 手算反向（链式法则） ----
# dL/da2 = -(y/a2 - (1-y)/(1-a2));  da2/dz2 = a2*(1-a2)
# 化简后：dL/dz2 = a2 - y
dz2 = a2 - y
dW2 = dz2 * a1
db2 = dz2
# 反传到 a1: dL/da1 = dz2 * W2
da1 = dz2 * W2
# 通过 sigmoid: dL/dz1 = da1 * a1*(1-a1)
dz1 = da1 * a1 * (1 - a1)
# 反传到 W1, b1
dW1 = dz1.unsqueeze(1) * x.unsqueeze(0)   # 外积
db1 = dz1
print(f'manual:   dW1.shape={list(dW1.shape)}  dW2.shape={list(dW2.shape)}')
```

```python
# ---- autograd 版 ----
W1a = W1.clone().requires_grad_()
b1a = b1.clone().requires_grad_()
W2a = W2.clone().requires_grad_()
b2a = b2.clone().requires_grad_()

z1a = W1a @ x + b1a
a1a = torch.sigmoid(z1a)
z2a = W2a @ a1a + b2a
a2a = torch.sigmoid(z2a)
loss_a = F.binary_cross_entropy(a2a, y)
loss_a.backward()

print('--- 梯度对比（手算 vs autograd 的最大绝对差） ---')
print(f'dW1:  {(dW1 - W1a.grad).abs().max().item():.2e}')
print(f'db1:  {(db1 - b1a.grad).abs().max().item():.2e}')
print(f'dW2:  {(dW2 - W2a.grad).abs().max().item():.2e}')
print(f'db2:  {(db2 - b2a.grad).abs().max().item():.2e}')
```

梯度应当在 $10^{-7}$ 量级一致——确认手算公式 $\frac{\partial \mathcal{L}}{\partial z_2} = \hat y - y$ 等推导无误。后续直接信任 `autograd`，不再手算。

> **数值稳定性提醒**：这里为了对照公式，故意用了非融合的 `sigmoid + binary_cross_entropy`，`a2` 接近 0 或 1 时 `log(a2)` 会溢出。实际训练中请用 `binary_cross_entropy_with_logits`——它在 logits 上用 log-sum-exp 技巧把 sigmoid 和 BCE 融合，数值上稳得多。

## 3. moons 数据集上的两层 MLP

两个交错的半月形——线性分类器分不开，最小可以用一个有少量隐藏神经元的 MLP 解决。

```python
from sklearn.datasets import make_moons as _sk_make_moons

def make_moons(n=400, noise=0.15, seed=0):
    X_np, y_np = _sk_make_moons(n_samples=n, noise=noise, random_state=seed)
    X = torch.tensor(X_np, dtype=torch.float32)
    y = torch.tensor(y_np, dtype=torch.float32).unsqueeze(1)
    return X, y

torch.manual_seed(0)
X_all, y_all = make_moons(n=400, noise=0.15)
split = 300
X_train, y_train = X_all[:split], y_all[:split]
X_test,  y_test  = X_all[split:], y_all[split:]

plt.scatter(X_train[y_train.squeeze()==0, 0], X_train[y_train.squeeze()==0, 1], s=8, label='class 0')
plt.scatter(X_train[y_train.squeeze()==1, 0], X_train[y_train.squeeze()==1, 1], s=8, label='class 1')
plt.legend(); plt.axis('equal'); plt.title('moons train'); plt.show()
```

### 模型 = `nn.Sequential` 拼 Linear + 激活

用 `BCEWithLogitsLoss`，模型最后一层只输出 logits，**不要再叠 sigmoid**——和 chap3 的 logistic 回归一样。

```python
torch.manual_seed(0)

model = nn.Sequential(
    nn.Linear(2, 8),
    nn.Tanh(),
    nn.Linear(8, 1),     # 输出 logits
)
loss_fn = nn.BCEWithLogitsLoss()
optimizer = optim.Adam(model.parameters(), lr=0.05)

history = []
for epoch in range(500):
    logits = model(X_train)
    loss = loss_fn(logits, y_train)
    optimizer.zero_grad(); loss.backward(); optimizer.step()
    history.append(loss.item())

with torch.no_grad():
    train_acc = ((torch.sigmoid(model(X_train)) >= 0.5).float() == y_train).float().mean()
    test_acc  = ((torch.sigmoid(model(X_test))  >= 0.5).float() == y_test ).float().mean()
print(f'final loss {history[-1]:.4f}  train_acc {train_acc.item():.4f}  test_acc {test_acc.item():.4f}')
```

## 4. 隐藏层尺寸 / 激活函数对比

做一个小实验：固定其他设置，改隐藏单元数 + 激活函数，看测试准确率。

```python
def fit_mlp(hidden, activation, epochs=400, seed=0):
    torch.manual_seed(seed)
    model = nn.Sequential(nn.Linear(2, hidden), activation(), nn.Linear(hidden, 1))
    opt = optim.Adam(model.parameters(), lr=0.05)
    loss_fn = nn.BCEWithLogitsLoss()
    for _ in range(epochs):
        loss = loss_fn(model(X_train), y_train)
        opt.zero_grad(); loss.backward(); opt.step()
    with torch.no_grad():
        acc = ((torch.sigmoid(model(X_test)) >= 0.5).float() == y_test).float().mean().item()
    return acc

print(f'{"hidden":>8s}  {"Tanh":>8s}  {"ReLU":>8s}  {"Sigmoid":>8s}')
for h in [2, 4, 8, 16, 32]:
    accs = {act_name: fit_mlp(h, act) for act_name, act in
            [('Tanh', nn.Tanh), ('ReLU', nn.ReLU), ('Sigmoid', nn.Sigmoid)]}
    print(f'{h:>8d}  {accs["Tanh"]:>8.4f}  {accs["ReLU"]:>8.4f}  {accs["Sigmoid"]:>8.4f}')
```

观察：

- 隐藏单元少（h=2）时容量不足，准确率低。
- 隐藏单元到 4-8 之后基本饱和，moons 比较简单。
- Sigmoid 在更深 / 更窄的网络里收敛慢（梯度消失），这里两层 + Adam 影响有限；后续 chap7 会进一步讨论。

## 5. 参数初始化：权重的高斯分布 vs 均匀分布

破解上面提到的“对称权重”最直接的办法就是随机初始化：给每个权重独立地采一个随机数，最常用的两种分布是**高斯分布**与**均匀分布**。下面直接用 `numpy` 各采 10000 个样本，画直方图看看两者的形状差异——这一段不依赖 `torch`，纯 `numpy` + `matplotlib` 就能跑。

```python
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(0)

# 高斯（正态）分布采样：loc 为均值、scale 为标准差
gaussian_weights = rng.normal(loc=0.0, scale=1.0, size=10000)
# 均匀分布采样：区间 [low, high)
uniform_weights = rng.uniform(low=-1.0, high=1.0, size=10000)

# 绘制两种参数分布
plt.figure(figsize=(9, 4))
plt.subplot(1, 2, 1)
plt.title('Gaussian Distribution')
plt.hist(gaussian_weights, bins=200, density=True)
plt.subplot(1, 2, 2)
plt.title('Uniform Distribution')
plt.hist(uniform_weights, bins=200, density=True)
plt.tight_layout()
plt.show()
```

高斯分布的权重集中在 0 附近、向两侧衰减，少数权重可能取到较大的绝对值；均匀分布则在 `[-1, 1)` 内大致等概率出现，没有“中间高两边低”的形状。两种采样都打破了对称性，让同一层不同神经元从不同的初值出发。

> **尺度才是关键**：实际训练里，比“选高斯还是均匀”更重要的是**方差（尺度）**的选取——方差太大容易使深层激活值爆炸、太小又会让信号逐层衰减。常用的 Xavier / He 初始化正是按每层的输入/输出维度自动缩放这个方差，PyTorch 的 `nn.Linear` 默认就采用了类似策略。

## 6. 优化问题：梯度消失与死亡 ReLU

上一节的随机初始化解决了对称权重，但**网络一深，新麻烦就来了**：反向传播时梯度要从输出端逐层相乘传回输入端，途中很容易被压成接近 0（梯度消失），或者被 ReLU 卡在负半轴永久归零（死亡 ReLU）。

下面用一个 **5 层前馈网络**（4 个隐藏层 + 1 个输出层）做两组对照小实验，把这两类失败模式「看得见」：每次只训练 **1 个回合**，反向后直接打印每层权重梯度的 $\ell_2$ 范数，跨层一比就一目了然。

为了排除「初始化没做好」这个干扰项，所有实验统一用 **He（Kaiming）初始化**（`std=sqrt(2/fan_in)`，专为 ReLU 设计、逐层保持前向方差稳定）。这样梯度若还衰减，锅就只能由激活函数来背。

> 本节直接复用 §3 的 moons 数据 `X_train, y_train`（2 维输入、二分类）。

```python
from functools import partial

# 把 torch 的就地初始化函数包成「接收一个张量即完成初始化」的可调用对象，
# 用法对齐书里的初始化器：Kaiming() / Constant(value=...)。
def Kaiming():
    # He（Kaiming）初始化：std = sqrt(2 / fan_in)，专为 ReLU 设计
    return partial(nn.init.kaiming_normal_, nonlinearity='relu')

def Constant(value=0.0):
    return partial(nn.init.constant_, val=value)


class Model_MLP_L5(nn.Module):
    """5 层前馈网络：4 个隐藏层(宽度 3) + 1 个输出层。
    通过 act 在 sigmoid / relu / lrelu 之间切换隐藏层激活；输出层固定用 sigmoid。
    权重默认 He 初始化，偏置默认置为 +1.0（与书一致，可按需改成大负数触发死亡 ReLU）。
    """
    def __init__(self, input_size, output_size,
                 act='sigmoid', w_init=None, b_init=None):
        super().__init__()
        self.fc1 = nn.Linear(input_size, 3)
        self.fc2 = nn.Linear(3, 3)
        self.fc3 = nn.Linear(3, 3)
        self.fc4 = nn.Linear(3, 3)
        self.fc5 = nn.Linear(3, output_size)
        if act == 'sigmoid':
            self.act = torch.sigmoid
        elif act == 'relu':
            self.act = F.relu
        elif act == 'lrelu':
            self.act = F.leaky_relu
        else:
            raise ValueError("act 只能是 'sigmoid' / 'relu' / 'lrelu'")
        # 默认值放在这里赋，避免可变默认参数在定义期被求值
        w_init = w_init if w_init is not None else Kaiming()
        b_init = b_init if b_init is not None else Constant(value=1.0)
        self.init_weights(w_init, b_init)

    def init_weights(self, w_init, b_init):
        for _, m in self.named_modules():
            if isinstance(m, nn.Linear):
                w_init(m.weight)
                b_init(m.bias)

    def forward(self, inputs):
        x = self.act(self.fc1(inputs))
        x = self.act(self.fc2(x))
        x = self.act(self.fc3(x))
        x = self.act(self.fc4(x))
        x = torch.sigmoid(self.fc5(x))   # 输出层固定 sigmoid，配 BCE 损失
        return x


def layer_grad_norms(model):
    """反向传播后，返回每个线性层权重梯度的 L2 范数 {层名: 范数}。
    用 isinstance(nn.Linear) 判定，和初始化遍历保持一致。"""
    norms = {}
    for name, m in model.named_modules():
        if isinstance(m, nn.Linear) and m.weight.grad is not None:
            norms[name] = m.weight.grad.norm(p=2).item()
    return norms


def one_step_grads(act='sigmoid', b_init=None, lr=0.01, seed=102):
    """建网 -> 单步前向/反向 -> 返回各层梯度范数。只跑 1 步即可看清逐层规律。"""
    torch.manual_seed(seed)
    model = Model_MLP_L5(input_size=2, output_size=1, act=act, b_init=b_init)
    optimizer = optim.SGD(model.parameters(), lr=lr)
    optimizer.zero_grad()
    y_hat = model(X_train)
    loss = F.binary_cross_entropy(y_hat, y_train)
    loss.backward()                 # 只反向、不 step，单步梯度足以看出衰减
    return layer_grad_norms(model)


print('helpers ready: Kaiming / Constant / Model_MLP_L5 / one_step_grads')
```

### 6.1 梯度消失：Sigmoid vs ReLU

Sigmoid 两端饱和、导数最大只有 0.25，逐层累乘后会把梯度不断压小。即便用了 He 初始化，5 层网络的梯度也会从输出层逐层衰减到输入端——靠近输入的参数几乎「收不到信号」。

把激活换成 **ReLU**（正区导数恒为 1）再跑一遍：配合 He 初始化逐层保持前向方差，各层梯度应当**保持在相近量级**，梯度消失基本消除。下面把两组逐层梯度范数打印出来并画到一张图上对比。

```python
# 同一个 5 层网络，分别用 Sigmoid 与 ReLU，单步反向后看各层梯度 L2 范数
sig_norms  = one_step_grads(act='sigmoid')
relu_norms = one_step_grads(act='relu')

layers = ['fc1', 'fc2', 'fc3', 'fc4', 'fc5']
print(f"{'layer':>5s}  {'sigmoid':>14s}  {'relu':>14s}")
for L in layers:
    print(f"{L:>5s}  {sig_norms[L]:>14.8g}  {relu_norms[L]:>14.8g}")

span_sig  = max(sig_norms.values())  / min(sig_norms.values())
span_relu = max(relu_norms.values()) / min(relu_norms.values())
print(f"\nsigmoid 跨度(最大/最小): {span_sig:8.1f}x  (~{math.log10(span_sig):.1f} 个数量级)")
print(f"relu    跨度(最大/最小): {span_relu:8.1f}x  (~{math.log10(span_relu):.1f} 个数量级)")

# 逐层梯度范数对比图（注意 y 轴用对数刻度）
plt.figure(figsize=(7, 4))
plt.plot(layers, [sig_norms[L]  for L in layers], 'o-', label='sigmoid')
plt.plot(layers, [relu_norms[L] for L in layers], 's-', label='relu')
plt.yscale('log')
plt.xlabel('layer'); plt.ylabel(r'$\ell_2$ norm of weight grad (log)')
plt.title('per-layer gradient norm: sigmoid vs relu')
plt.legend(); plt.grid(alpha=0.3, which='both'); plt.show()
```

### 6.2 死亡 ReLU：被大负偏置「杀死」，再用 Leaky ReLU 救活

ReLU 缓解了梯度消失，但带来新风险：$x<0$ 时 ReLU 输出和导数都恒为 0。若某次更新让一个神经元在**所有训练样本**上的输入都跌到 0 以下，它的梯度就永远停在 0、再也无法激活——相当于被永久「杀死」。

下面**故意**把偏置初始化成一个较大的负数 `Constant(value=-8.0)`，让线性输出大概率落入负半轴，触发死亡 ReLU；然后只把激活换成 **Leaky ReLU**（负半轴保留 0.01 的小斜率），看能否把网络从死局里救回来。

```python
# 偏置故意置大负数 -> ReLU 输入大概率落负半轴 -> 触发死亡 ReLU
dead_norms  = one_step_grads(act='relu',  b_init=Constant(value=-8.0))
# 仅把激活换成 Leaky ReLU，其它配置完全相同
revive_norms = one_step_grads(act='lrelu', b_init=Constant(value=-8.0))

print(f"{'layer':>5s}  {'relu (dead)':>14s}  {'lrelu (revived)':>16s}")
for L in layers:
    print(f"{L:>5s}  {dead_norms[L]:>14.8g}  {revive_norms[L]:>16.8g}")

all_dead = all(v == 0.0 for v in dead_norms.values())
print(f"\nReLU 各层梯度是否全部精确为 0（网络死透）: {all_dead}")
print("Leaky ReLU 把梯度从全零恢复成非零，但仍沿反向逐层衰减"
      f"（fc5 {revive_norms['fc5']:.3g} -> fc1 {revive_norms['fc1']:.3g}）")
```

**小结（优化问题）**

- **梯度消失**：Sigmoid 两端饱和、导数 $\le 0.25$，深网络里逐层累乘把梯度压小；即便用 He 初始化，5 层网仍跨约 2.5 个数量级。换成 **ReLU**（正区导数恒为 1）+ He 初始化后，各层梯度保持在相近量级，梯度消失基本消除。
- **死亡 ReLU**：把偏置初始化成大负数，ReLU 神经元被卡在负半轴，**所有层梯度精确为 0**、网络再也学不动。换成 **Leaky ReLU**（负半轴留 0.01 斜率）即可把梯度从全零救活，但仍残留逐层衰减。
- **进一步**：想再削弱衰减，可调大 Leaky ReLU 的负斜率，或改用 ELU / GELU / Swish 等更平滑的激活函数。这也是后面深层网络普遍弃用 Sigmoid 隐藏层的根本原因。

## 小结

- PyTorch 的训练循环固定模板：`logits = model(x); loss = loss_fn(logits, y); opt.zero_grad(); loss.backward(); opt.step()`。
- **logits-only 约定**：分类时模型最后一层只输出 logits，`BCEWithLogitsLoss` / `CrossEntropyLoss` 内部融合 sigmoid/softmax + 交叉熵，数值更稳。
- **autograd** 通过链式法则自动反传梯度——本节通过手算验证了 PyTorch 算出的梯度与公式一致。
- `nn.Sequential` 是最轻量的网络组装方式；更复杂的结构用 `nn.Module` 子类（chap4 下示范）。
