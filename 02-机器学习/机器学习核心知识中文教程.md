# 机器学习核心知识中文教程

这篇把国内外强教程的主干整理成中文学习版。你先读这个，再去 GitHub 原仓库跑代码。

来源参考：Microsoft ML-For-Beginners、scikit-learn 官方文档、Hands-On ML、Datawhale 南瓜书、Made With ML、Kaggle Learn。

## 1. 机器学习的标准流程

```mermaid
graph LR
    A[定义问题] --> B[收集数据]
    B --> C[数据清洗]
    C --> D[特征工程]
    D --> E[切分数据]
    E --> F[训练基线模型]
    F --> G[评估模型]
    G --> H[调参和对比]
    H --> I[错误分析]
    I --> J[保存和部署]
```

每个项目都按这个流程走。不要一上来追复杂模型。

## 2. 监督学习

监督学习有标签。模型学习“输入特征 → 正确答案”的关系。

### 2.1 回归

回归预测连续数值。

典型任务：

- 房价预测；
- 销量预测；
- 广告消耗预测；
- 气温预测。

常用模型：

| 模型 | 适合什么 | 优点 | 缺点 |
|---|---|---|---|
| 线性回归 | 关系接近线性的数据 | 简单、可解释 | 表达能力弱 |
| Ridge / Lasso | 特征较多、需要正则化 | 减少过拟合 | 仍然偏线性 |
| 决策树回归 | 非线性表格数据 | 容易解释 | 容易过拟合 |
| 随机森林回归 | 表格数据强基线 | 稳定、少调参 | 模型较大 |
| 梯度提升树 | 结构化数据竞赛 | 效果强 | 调参更复杂 |

常用指标：

| 指标 | 中文意思 | 什么时候看 |
|---|---|---|
| MAE | 平均绝对误差 | 想要直观误差 |
| MSE | 平均平方误差 | 惩罚大错误 |
| RMSE | 均方根误差 | 和标签同单位 |
| $R^2$ | 解释方差比例 | 看整体拟合程度 |

### 2.2 分类

分类预测类别。

典型任务：

- 是否垃圾邮件；
- 用户是否流失；
- 图片类别；
- 疾病风险等级；
- 广告是否转化。

常用模型：

| 模型 | 适合什么 | 优点 | 注意 |
|---|---|---|---|
| Logistic 回归 | 二分类基线 | 简单、输出概率 | 需要特征处理 |
| KNN | 小数据、边界简单 | 思路直观 | 数据大时慢 |
| 决策树 | 需要解释规则 | 可视化强 | 容易过拟合 |
| 随机森林 | 表格分类 | 稳定强基线 | 解释性下降 |
| 朴素贝叶斯 | 文本分类 | 快、简单 | 独立性假设强 |
| SVM | 中小规模复杂边界 | 间隔思想强 | 大数据慢 |
| 梯度提升树 | 表格任务 | 效果强 | 调参成本高 |

分类指标：

| 指标 | 解释 | 适合场景 |
|---|---|---|
| accuracy | 总体预测对的比例 | 类别均衡 |
| precision | 预测为正的有多少是真的 | 误报成本高 |
| recall | 真实为正的找回多少 | 漏报成本高 |
| F1 | precision 和 recall 的折中 | 类别不均衡 |
| ROC-AUC | 排序能力 | 二分类模型比较 |

例子：欺诈识别更重视 recall，因为漏掉欺诈很贵；营销触达更重视 precision，因为误触达会浪费预算。

## 3. 无监督学习

无监督学习没有标签，重点是发现结构。

| 方法 | 解决什么问题 | 常见用法 |
|---|---|---|
| K-Means | 把样本分成 K 类 | 用户分群 |
| DBSCAN | 找密度相近的群和噪声 | 异常点发现 |
| PCA | 降维 | 可视化、去冗余 |
| t-SNE / UMAP | 高维数据可视化 | 看嵌入空间结构 |
| Isolation Forest | 异常检测 | 风控、运维 |

无监督结果不能只看分数，要结合业务解释。

## 4. 特征工程

模型效果经常不是由模型决定，而是由数据和特征决定。

| 数据类型 | 常见处理 |
|---|---|
| 数值特征 | 缺失值填充、标准化、异常值处理 |
| 类别特征 | one-hot、目标编码、频次编码 |
| 时间特征 | 年月日、星期、节假日、时间差 |
| 文本特征 | 分词、TF-IDF、Embedding |
| 图像特征 | 像素、预训练模型特征 |

新手先掌握：

1. 缺失值；
2. 类别编码；
3. 标准化；
4. 训练/测试一致处理；
5. 防止数据泄漏。

## 5. 数据泄漏

数据泄漏是新手最常见的大坑：训练时用了未来或测试集信息。

常见泄漏：

- 先对全量数据标准化，再切分训练/测试；
- 特征里包含标签的变体；
- 用未来数据预测过去；
- 调参时反复看测试集；
- 用户 ID、订单号这类字段暗含答案。

正确做法：

1. 先切分数据；
2. 只在训练集上 `fit` 预处理器；
3. 验证集用来调参；
4. 测试集最后只用一次。

## 6. 基线模型

基线模型是最简单可运行方案。

| 任务 | 基线 |
|---|---|
| 回归 | 预测均值、线性回归 |
| 二分类 | 预测多数类、Logistic 回归 |
| 多分类 | Logistic/Softmax、随机森林 |
| 表格数据 | 随机森林、梯度提升树 |
| 文本分类 | TF-IDF + Logistic/朴素贝叶斯 |

没有基线就不知道复杂模型是不是真的有提升。

## 7. 调参顺序

不要先调模型，先看数据。

推荐顺序：

1. 明确指标；
2. 做好数据切分；
3. 跑简单基线；
4. 查错误样本；
5. 改特征；
6. 换模型；
7. 最后调参。
