## 一、大语言模型在投流中的应用

### 1.1 LLM基础技术

**【1】Transformer架构原理**
- 核心要点：Transformer是投流Agent的核心技术基础，采用自注意力机制（Self-Attention）处理序列数据。其核心优势在于能够并行处理所有位置的信息，相比RNN/LSTM具有更强的长距离依赖建模能力。
- 技术细节：多头注意力机制（Multi-Head Attention）允许模型同时关注不同位置的不同表示子空间；位置编码（Positional Encoding）解决序列顺序问题；层归一化（Layer Normalization）稳定训练过程。
- 投流应用：用于理解广告文案语义、生成创意素材、分析用户行为序列、预测转化概率。

**【2】GPT系列模型演进**
- 核心要点：GPT-1到GPT-4的演进展示了大模型在投流领域的应用潜力。GPT-3的175B参数规模使其具备少样本学习能力，GPT-4的多模态能力支持图文联合分析。
- 技术细节：GPT-3.5引入RLHF（人类反馈强化学习）提升输出质量；GPT-4支持128K上下文窗口，可处理长文档；GPT-4o实现文本、图像、音频统一处理。
- 投流应用：广告文案生成、用户意图理解、竞品分析、市场趋势预测。

**【3】国产大模型技术特点**
- 核心要点：百度文心一言、阿里通义千问、腾讯混元、字节豆包等国产大模型在中文理解和生成方面具有优势，更适合国内投流场景。
- 技术细节：文心一言采用ERNIE架构，强化中文语义理解；通义千问支持10万+上下文；混元擅长多模态内容生成；豆包针对抖音生态优化。
- 投流应用：中文广告创意生成、本地化内容理解、平台生态适配。

**【4】垂直领域大模型**
- 核心要点：针对广告投放领域训练的垂直大模型，如广告文案生成模型、创意评估模型、用户画像模型等，在特定任务上表现优于通用大模型。
- 技术细节：通过领域数据微调（Fine-tuning）、指令微调（Instruction Tuning）、人类反馈强化学习（RLHF）等方式训练；采用LoRA、QLoRA等参数高效微调技术降低成本。
- 投流应用：专业广告文案生成、精准用户匹配、效果预测优化。

**【5】多模态大模型**
- 核心要点：能够同时处理文本、图像、视频、音频等多种模态的大模型，为投流Agent提供更全面的素材分析和生成能力。
- 技术细节：采用跨模态注意力机制、对比学习、模态对齐等技术；支持图像描述生成、视频内容理解、音频情感分析等任务。
- 投流应用：广告素材多维分析、视频内容理解、跨模态创意生成。

### 1.2 Prompt工程技术

**【6】系统提示词设计**
- 核心要点：为投流Agent设计高质量的系统提示词（System Prompt），定义其角色、能力和行为边界。好的系统提示词应包含角色定义、任务说明、输出格式、约束条件等要素。
- 实践建议：明确Agent的专业领域（如"你是一位资深的信息流广告优化师"）；设定行为准则（如"所有建议必须基于数据"）；定义输出格式（如"以表格形式呈现关键指标"）。
- 案例示例：设计一个专门用于抖音投流的Agent提示词，包含平台规则、出价策略、素材要求等约束。

**【7】少样本学习（Few-shot Learning）**
- 核心要点：通过在提示词中提供少量示例，引导大模型理解任务模式并生成符合要求的输出。在投流场景中，可用于广告文案生成、效果预测等任务。
- 实践建议：选择3-5个高质量示例；示例应覆盖不同场景和难度；保持示例格式一致；定期更新示例库。
- 案例示例：提供5个成功的广告文案示例，让Agent学习文案风格和结构。

**【8】思维链提示（Chain-of-Thought）**
- 核心要点：引导大模型逐步推理复杂问题，通过展示思考过程提升决策质量。在投流优化中，可用于分析投放数据、制定优化策略等复杂任务。
- 实践建议：在提示词中加入"让我们一步步分析"等引导语；要求模型展示推理过程；对复杂问题进行分步骤处理。
- 案例示例：分析某广告计划ROI下降的原因，要求Agent逐步列出可能原因并分析。

**【9】结构化输出控制**
- 核心要点：通过提示词约束大模型的输出格式，使其生成结构化的数据（如JSON、表格），便于后续程序处理。在投流Agent中，用于生成标准化的投放报告、策略方案等。
- 实践建议：明确指定输出格式（如"以JSON格式返回"）；提供格式模板；使用正则表达式验证输出；处理格式异常。
- 案例示例：要求Agent以特定JSON格式返回广告计划的优化建议。

**【10】提示词模板管理**
- 核心要点：建立提示词模板库，对不同任务场景的提示词进行分类管理和版本控制。提升提示词复用性，降低维护成本。
- 实践建议：按任务类型分类（文案生成、数据分析、策略制定等）；建立版本控制机制；定期评估和优化模板；支持A/B测试。
- 工具推荐：LangSmith、PromptFlow、Anthropic Prompt Library等。

### 1.3 检索增强生成（RAG）

**【11】RAG架构原理**
- 核心要点：检索增强生成（Retrieval-Augmented Generation）通过检索外部知识库增强大模型的回答能力。在投流Agent中，RAG可用于查询历史投放数据、行业报告、最佳实践等。
- 技术架构：文档分块（Chunking）→ 向量化（Embedding）→ 索引存储（Vector Store）→ 检索（Retrieval）→ 生成（Generation）。
- 投流应用：查询历史投放案例、获取行业基准数据、检索广告平台规则、参考最佳实践。

**【12】文档分块策略**
- 核心要点：将长文档切分为适合检索的小块，是RAG效果的关键。分块策略包括固定长度分块、语义分块、递归分块等。
- 技术细节：固定长度分块简单但可能破坏语义；语义分块基于主题边界切分；递归分块逐层细化；重叠分块保留上下文。
- 实践建议：根据文档类型选择策略；保持块大小适中（200-500 tokens）；添加适当的重叠；保留元数据。

**【13】向量化模型选择**
- 核心要点：选择合适的向量化模型（Embedding Model）将文本转换为向量表示，影响检索的准确性和效率。
- 主流模型：OpenAI text-embedding-3-small/large、BGE系列、M3E、text2vec-chinese等。
- 选择标准：中文支持能力、向量维度、推理速度、成本、开源/闭源等。
- 投流应用：中文广告语义理解、用户意图匹配、创意素材相似度计算。

**【14】向量数据库选型**
- 核心要点：向量数据库用于存储和检索向量化后的文档，是RAG系统的核心组件。
- 主流方案：Pinecone（云原生）、Milvus（开源）、Weaviate（开源）、Chroma（轻量级）、FAISS（Facebook开源）等。
- 选择标准：数据规模、查询性能、部署方式、成本、生态支持等。
- 投流应用：存储和检索海量投放数据、用户行为日志、广告素材库。

**【15】混合检索策略**
- 核心要点：结合稠密检索（向量相似度）和稀疏检索（关键词匹配）的优势，提升检索效果。在投流场景中，混合检索可同时处理语义匹配和精确关键词查询。
- 技术实现：BM25关键词检索 + 向量语义检索 → 结果融合（RRF/加权）→ 重排序。
- 实践建议：根据查询类型动态调整权重；使用重排序模型提升结果质量；评估不同组合的效果。

**【16】检索优化技巧**
- 核心要点：通过查询改写、结果过滤、重排序等技术优化RAG检索效果。
- 查询改写：使用大模型改写用户查询，提升召回率；添加上下文扩展查询；生成假设性答案辅助检索。
- 结果过滤：基于相关性阈值过滤低质量结果；基于时效性过滤过期信息；基于来源可信度过滤。
- 重排序：使用交叉编码器（Cross-Encoder）对检索结果重新排序；结合多维度特征综合评分。

**【17】RAG评估方法**
- 核心要点：评估RAG系统的效果，包括检索质量和生成质量两个维度。
- 检索评估指标：召回率（Recall）、准确率（Precision）、MRR、NDCG等。
- 生成评估指标：忠实度（Faithfulness）、相关性（Relevance）、完整性（Completeness）等。
- 评估工具：RAGAS、TruLens、LangSmith等。

**【18】多模态RAG**
- 核心要点：扩展传统文本RAG到多模态领域，支持图像、视频等非文本内容的检索和生成。在投流场景中，可用于检索相似广告素材、分析竞品创意等。
- 技术实现：多模态向量化（CLIP、BLIP等）→ 多模态向量数据库 → 跨模态检索 → 多模态生成。
- 投流应用：检索相似广告素材、分析竞品创意、生成多模态广告内容。

**【19】知识图谱增强RAG**
- 核心要点：将知识图谱与RAG结合，利用结构化知识增强检索和推理能力。在投流场景中，可构建广告投放知识图谱，包含平台、行业、策略、效果等实体及其关系。
- 技术实现：知识图谱构建 → 实体链接 → 子图检索 → 图增强生成。
- 投流应用：理解广告投放领域的复杂关系、推理优化策略、解释决策依据。

**【20】Agentic RAG**
- 核心要点：将RAG与Agent能力结合，使系统能够自主决定何时检索、检索什么、如何使用检索结果。在投流场景中，Agent可根据任务需求自主调用不同的知识源。
- 技术实现：任务分解 → 检索决策 → 多源检索 → 结果整合 → 生成回答。
- 投流应用：智能问答系统、自动研究报告生成、动态策略推荐。

### 1.4 强化学习应用

**【21】强化学习基础**
- 核心要点：强化学习（Reinforcement Learning, RL）是投流Agent实现自主优化的核心技术。Agent通过与环境（广告平台）交互，学习最优的投放策略以最大化奖励（ROI/转化）。
- 核心概念：状态（State）、动作（Action）、奖励（Reward）、策略（Policy）、价值函数（Value Function）。
- 投流映射：状态=当前投放数据和用户特征；动作=出价调整、创意更换、定向修改等；奖励=转化收益或ROI提升。

**【22】Q-Learning在投流中的应用**
- 核心要点：Q-Learning是一种无模型的强化学习算法，通过学习动作价值函数（Q函数）来选择最优动作。在投流中，可用于学习不同情境下的最优出价策略。
- 技术细节：Q表维护状态-动作对的价值；通过贝尔曼方程更新Q值；使用ε-贪心策略平衡探索与利用。
- 投流应用：学习不同用户群体的最优出价、不同时间段的预算分配、不同创意的投放策略。

**【23】深度Q网络（DQN）**
- 核心要点：使用深度神经网络近似Q函数，解决高维状态空间问题。在投流中，可处理复杂的用户特征和投放环境。
- 技术改进：经验回放（Experience Replay）打破数据相关性；目标网络（Target Network）稳定训练；双DQN减少过估计。
- 投流应用：处理高维用户特征、学习复杂投放策略、适应动态变化的市场环境。

**【24】策略梯度方法**
- 核心要点：直接优化策略函数，适用于连续动作空间或随机策略场景。在投流中，可用于学习概率化的出价策略或创意选择策略。
- 主流算法：REINFORCE、A2C、PPO、SAC等。
- 投流应用：学习概率化出价策略、多目标优化、连续动作空间的预算分配。

**【25】多臂老虎机（MAB）**
- 核心要点：多臂老虎机是强化学习的简化形式，适用于探索-利用权衡问题。在投流中，可用于广告创意测试、出价策略探索等场景。
- 算法变体：ε-贪心、UCB（置信上界）、Thompson Sampling等。
- 投流应用：A/B测试自动优化、创意探索与利用、动态预算分配。

**【26】上下文多臂老虎机（Contextual Bandit）**
- 核心要点：在多臂老虎机基础上引入上下文信息，根据当前状态选择最优动作。在投流中，可根据用户特征、时间、平台等因素动态调整策略。
- 技术实现：LinUCB、Neural Bandit等算法；特征工程与模型结合。
- 投流应用：个性化出价、动态创意选择、实时策略调整。

**【27】离线强化学习**
- 核心要点：从历史数据中学习策略，无需在线交互。在投流中，可利用历史投放数据训练策略，降低在线探索成本和风险。
- 技术挑战：分布偏移、外推误差、保守估计等。
- 主流算法：BCQ、CQL、IQL等。
- 投流应用：从历史数据学习最佳实践、安全地测试新策略、降低冷启动成本。

**【28】多智能体强化学习（MARL）**
- 核心要点：多个Agent协同学习和决策。在投流中，可用于多账户协同优化、跨平台联合投放、团队协作等场景。
- 技术挑战：信用分配、非平稳环境、通信效率等。
- 主流算法：QMIX、MAPPO、CommNet等。
- 投流应用：多账户协同优化、跨平台联合投放、预算动态分配。

**【29】奖励塑造（Reward Shaping）**
- 核心要点：设计合适的奖励函数引导Agent学习期望行为。在投流中，奖励函数设计直接影响策略质量。
- 设计原则：奖励应与业务目标对齐；避免奖励欺骗；平衡短期和长期目标；考虑多目标权衡。
- 投流应用：平衡ROI和规模、考虑用户生命周期价值、处理延迟转化。

**【30】探索策略设计**
- 核心要点：设计有效的探索策略，平衡探索新策略和利用已知最优策略。在投流中，探索是发现更好策略的关键，但过度探索会浪费预算。
- 探索方法：ε-贪心、玻尔兹曼探索、UCB、Thompson Sampling、噪声网络等。
- 投流应用：新创意测试、新人群探索、新出价策略尝试。

### 1.5 自然语言处理技术

**【31】文本分类在投流中的应用**
- 核心要点：文本分类技术用于对广告文案、用户评论、搜索词等文本进行分类，支持投放策略制定。
- 应用场景：广告文案质量评估、用户意图分类、情感分析、主题分类、行业分类等。
- 技术方案：传统机器学习（SVM、朴素贝叶斯）、深度学习（TextCNN、BERT）、大模型微调。

**【32】命名实体识别（NER）**
- 核心要点：识别文本中的实体（品牌、产品、地名等），在投流中可用于提取广告文案中的关键信息、分析竞品提及等。
- 应用场景：竞品分析、品牌监测、产品特征提取、地名识别（本地投放）等。
- 技术方案：BiLSTM-CRF、BERT-NER、大模型few-shot NER。

**【33】文本相似度计算**
- 核心要点：计算文本之间的语义相似度，在投流中可用于去重、相似创意检测、用户意图匹配等。
- 应用场景：广告去重、创意库管理、搜索词匹配、用户意图理解等。
- 技术方案：余弦相似度、编辑距离、语义嵌入相似度、SimCSE等。

**【34】情感分析**
- 核心要点：分析文本的情感倾向，在投流中可用于评估用户反馈、分析评论情感、监测品牌声誉等。
- 应用场景：用户评论分析、社交媒体监测、广告效果评估、危机预警等。
- 技术方案：基于词典、基于机器学习、基于深度学习、基于大模型。

**【35】自动摘要**
- 核心要点：自动生成文本摘要，在投流中可用于生成投放报告摘要、竞品分析摘要、行业趋势摘要等。
- 应用场景：投放报告自动生成、长文档摘要、会议纪要生成等。
- 技术方案：抽取式摘要、生成式摘要、大模型摘要。

**【36】文本生成**
- 核心要点：自动生成文本内容，在投流中是创意生成的核心技术。
- 应用场景：广告文案生成、落地页文案、推送通知、邮件营销等。
- 技术方案：模板填充、序列到序列模型、GPT系列、大模型微调。

**【37】对话系统**
- 核心要点：构建智能对话系统，在投流中可用于客户服务、销售咨询、投放指导等场景。
- 应用场景：广告投放智能客服、优化师助手、用户咨询服务等。
- 技术方案：检索式对话、生成式对话、任务型对话、大模型对话。

**【38】信息抽取**
- 核心要点：从非结构化文本中抽取结构化信息，在投流中可用于提取广告规则、竞品信息、行业数据等。
- 应用场景：广告规则提取、竞品信息结构化、行业数据整理等。
- 技术方案：关系抽取、事件抽取、大模型信息抽取。

**【39】知识抽取与图谱构建**

### 1.8 预测模型技术

**【61】点击率预估（CTR）**
- 核心要点：预测广告被点击的概率，是广告投放的核心技术。在投流中用于出价决策、广告排序等。
- 技术演进：LR → GBDT → FM → DeepFM → 动态特征模型。
- 关键特征：用户特征、广告特征、上下文特征、交叉特征等。
- 评估指标：AUC、LogLoss、GAUC等。

**【62】转化率预估（CVR）**
- 核心要点：预测广告点击后转化为目标行为的概率。在投流中用于oCPC/oCPM出价。
- 技术挑战：样本选择偏差、数据稀疏、延迟转化等。
- 解决方案：ESMM、多任务学习、延迟转化建模等。

**【63】用户生命周期价值预测（LTV）**
- 核心要点：预测用户的长期价值，在投流中用于高价值用户获取和出价优化。
- 应用场景：高价值用户定向、出价上限设定、预算分配优化等。
- 技术方案：生存分析、序列模型、增量模型等。

**【64】预算消耗预测**
- 核心要点：预测广告预算的消耗速度和效果，在投流中用于预算控制和分配。
- 应用场景：预算 pacing、消耗预警、效果预测等。
- 技术方案：时间序列模型、生存分析、深度学习等。

**【65】归因建模**
- 核心要点：分析各营销触点对转化的贡献，在投流中用于评估广告效果和预算分配。
- 归因方法：末次点击、首次点击、线性归因、时间衰减、数据驱动归因等。
- 技术方案：Shapley值、马尔可夫链、深度学习归因等。

**【66】增量建模**
- 核心要点：评估广告的真实增量效果，区分自然转化和广告带来的转化。
- 应用场景：广告效果真实评估、预算优化、渠道选择等。
- 技术方案：Uplift Modeling、因果森林、T-Learner、S-Learner等。

**【67】异常检测**
- 核心要点：检测投放数据中的异常情况，在投流中用于风控和质量控制。
- 异常类型：流量异常、转化异常、素材违规、账户异常等。
- 技术方案：统计方法、Isolation Forest、Autoencoder、时序异常检测等。

**【68】时序预测**
- 核心要点：基于历史数据预测未来趋势，在投流中用于效果预测、预算规划等。
- 应用场景：投放效果预测、预算消耗预测、市场趋势预测等。
- 技术方案：ARIMA、Prophet、LSTM、Transformer等。

**【69】因果推断**
- 核心要点：分析变量间的因果关系，在投流中用于评估广告的真实效果。
- 应用场景：广告效果评估、策略优化、用户分群等。
- 技术方案：随机实验、倾向得分匹配、双重差分、工具变量等。

**【70】多目标优化**
- 核心要点：同时优化多个目标，在投流中用于平衡ROI、规模、质量等多个指标。
- 应用场景：多目标出价、预算分配、创意优化等。
